Technical Architecture

Tech Stack

  • Backend: Python Flask (lightweight web framework)
  • API Integration: GLPI REST API with custom GLPIClient class
  • Data Visualization: Chart.js for real-time charts and metrics
  • Frontend: Bootstrap 5 (responsive, mobile-ready design)
  • Deployment: Docker containerized with Gunicorn WSGI server
  • Web Server: Nginx reverse proxy (production-ready configuration)

Backend Architecture

The Flask backend is built around a custom GLPIClient class that handles authentication, session management, and API communication with the GLPI instance. It exposes 7 REST API endpoints that power the dashboard:

1
/api/priority-tickets

Returns today's high-priority tickets with status, assigned tech, and time-to-close metrics. Filters tickets by urgency level and creation date.

2
/api/onboarding

Tracks employee onboarding and offboarding workflows. Returns active workflows with completion percentage, assigned IT staff, and pending tasks. Currently tracking Dr. Emily Sanchez's onboarding with Google Workspace setup, equipment provisioning, and access grants.

3
/api/activity-feed

Aggregates recent ticket activity (new tickets, status changes, assignments, resolutions) into a unified timeline. Provides real-time operational visibility across all IT service categories.

4
/api/security-alerts

Filters security and incident tickets (HIPAA breaches, phishing attempts, unauthorized access). Includes severity classification and time-since-detection metrics.

5
/api/system-health

Monitors infrastructure health based on open incident tickets. Tracks network uptime, server status, EHR availability, and critical system alerts.

6
/api/assets

Provides asset inventory summary (computers, monitors, network equipment, medical devices). Returns asset counts by type, condition, and deployment status. Currently tracking 140+ assets across Northwoods Health System.

7
/api/sla-metrics

Calculates SLA performance metrics based on ticket resolution times. Tracks response time goals, resolution time goals, and percentage of tickets meeting SLA targets by priority level.

Technical Highlights

GLPIClient Architecture

The custom GLPIClient class encapsulates all GLPI API interactions with intelligent session management:

  • Automatic session initialization: Authenticates with GLPI API using user token and app token
  • Session reuse: Maintains active session across requests to minimize authentication overhead
  • Graceful error handling: Catches API errors, timeouts, and connection failures without crashing the dashboard
  • Flexible querying: Supports complex GLPI search criteria (filtering by date, status, category, urgency, etc.)
  • Data transformation: Converts GLPI's raw API responses into clean, dashboard-ready JSON

Chart.js Data Visualization

All metrics are visualized with Chart.js for immediate operational insight:

  • Ticket status distribution: Doughnut charts showing open/assigned/resolved breakdown
  • SLA performance trends: Bar charts tracking resolution time compliance by priority level
  • Workload distribution: Horizontal bar charts showing ticket assignments across IT staff
  • System health indicators: Color-coded status badges (green/yellow/red) for critical infrastructure

Auto-Refresh Implementation

The dashboard uses JavaScript's setInterval() to poll all 7 API endpoints every 60 seconds. When new data arrives, Chart.js animations smoothly update visualizations without jarring full-page reloads. Users can monitor operations continuously without interaction.
